Public Methods

setActiveSlide

since v0.0.1 
          
            <script>
              (async () => {
                await customElements.whenDefined('safety-slider');
                const safetySliderElement = document.querySelector('safety-slider');
                await safetySliderElement.setActiveSlide(1);
              })();
            </script>

Browser Support

Safety Slider is built with the StencilJS toolchain and supports all natively supported browsers for Stencil. The only browsers excluded are Internet Explorer and Legacy Edge.


For more information see Stencil's browser support page .


Integrations

Website integration

safety-slider has files specifically generated for website usage in the www/build directory. Place those files together where you would like them to be, then add the following script tags to the end of the HTML body tag. 

          
            <script type="module" src="/path-to-files/safety-slider.esm.js"></script>
            <script nomodule src="/path-to-files/safety-slider.js"></script>

Styling

The styling of safety-slider is the bare minimum. This is intentional as many will prefer to add their own styling anyway.

Custom styling can be easily achieved by changing the values of the CSS variables used within the slider. See the example below.

          
            <style>
              safety-slider.custom-styling-example {
                --safety-slider-btn-color: darkgreen;
                --safety-slider-btn-disabled-color: darkseagreen;
              }

              safety-slider.custom-styling-example safety-slider-arrows {
                --safety-slider-arrow-border-width: 3px;
              }
            </style>

            <safety-slider class="custom-styling-example">
              ...
            </safety-slider>

Below lists all of the available CSS variables that can be altered to fit a custom style.

          
            // Scoped to <safety-slider>
            --safety-slider-bottom-padding: 2rem;
            --safety-slider-x-padding: 3rem;
            --safety-slider-btn-color: black;
            --safety-slider-btn-disabled-color: gainsboro;

            // Scoped to <safety-slider-arrows>
            --safety-slider-arrow-border-width: 1px;
            --safety-slider-arrow-border-color: var(--safety-slider-btn-color);
            --safety-slider-arrow-background-color: white;
            --safety-slider-arrow-padding: 0;
            --safety-slider-arrow-color: var(--safety-slider-btn-color);
            --safety-slider-arrow-width: 2rem;
            --safety-slider-arrow-height: var(--safety-slider-arrow-width);
            --safety-slider-arrow-disabled-background-color: var(--safety-slider-btn-disabled-color);

            // Scoped to <safety-slider-dots>
            --safety-slider-dots-container-height: calc(var(--safety-slider-bottom-padding) / 2);
            --safety-slider-dots-container-position-bottom: calc(var(--safety-slider-bottom-padding) / 8);
            --safety-slider-dot-background-color: var(--safety-slider-btn-color);
            --safety-slider-dot-width: .75rem;
            --safety-slider-dot-height: var(--safety-slider-dot-width);
            --safety-slider-dot-x-margin: calc(var(--safety-slider-dot-width) / 2);
            --safety-slider-dot-disabled-background-color: var(--safety-slider-btn-disabled-color);

The CSS variables only encompass styling properties that are already defined by safety-slider's minimum styling. Any properties beyond them can simply be added by creating a rule using that element's class as a selector. For example, arrow buttons use safety-slider-arrow and dot buttons use safety-slider-dot.
